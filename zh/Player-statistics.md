# 玩家统计

Empostor 可以追踪每位玩家的游戏统计数据，包括胜场、击杀、完成任务数等。统计数据可通过管理面板和游戏内 `/stat` 命令查看。

此功能**默认禁用**。在 `config.json` 中启用以开始追踪。

## 配置

将 `PlayerStats` 部分添加到 `config.json`：

```json
{
  "PlayerStats": {
    "Enabled": false,
    "PersistToFile": true
  }
}
```

| 键 | 默认值 | 描述 |
| :--- | :--- | :--- |
| **Enabled** | `false` | 是否启用玩家统计追踪。 |
| **PersistToFile** | `true` | 是否将统计数据保存到磁盘上的 `player_stats.json`。如果禁用，统计数据仅保存在内存中，服务器重启后丢失。 |

## 追踪的统计数据

以下指标按玩家记录（以好友代码为键）：

| 统计项 | 描述 |
| :--- | :--- |
| 游戏场次 | 玩家参与的游戏总场次 |
| 胜场 | 作为船员获胜的场次 |
| 败场 | 作为船员失败的场次 |
| 内鬼胜场 | 作为内鬼获胜的场次 |
| 击杀 | 总击杀数 |
| 死亡 | 总死亡次数 |
| 完成任务 | 完成的任务总数 |
| 被投票出局 | 被投票出局的次数 |

## 管理面板

启用后，管理面板的监控部分下会出现**统计**标签页。显示按游戏场次排序的排名表格，包含所有统计数据的列。使用**刷新**按钮重新加载数据，使用**全部重置**清除所有统计（需要确认）。

## 游戏内命令

玩家可以在游戏内聊天中输入 `/stat`（或 `/stats`、`/mystats`）查看自己的统计。回复以私密方式发送，并根据玩家语言进行本地化。

## 本地化

`/stat` 命令回复支持 Empstor 语言系统中配置的所有语言。如果玩家语言没有对应的翻译，则使用英语作为后备。

## API 端点

管理面板使用以下 API 端点：

| 方法 | 路径 | 描述 |
| :--- | :--- | :--- |
| `GET` | `/api/admin/player/stats` | 返回所有玩家统计数据，按游戏场次排序。响应：`{"enabled": true, "players": [...]}` |
| `GET` | `/api/admin/player/stats/{friendCode}` | 按好友代码返回单个玩家的统计数据。 |
| `POST` | `/api/admin/player/stats/reset` | 清除所有玩家统计数据。需要管理员认证。 |

## 文件持久化

当 `PersistToFile` 启用时，统计数据保存到服务器目录中的 `player_stats.json`。文件在每场游戏结束后更新，与封禁和举报存储的模式相同。
